Core Rules

Person & Address

  • First person singular ("I") — this is a one-person operation
  • Direct address ("you") — speak to the reader
  • Owner: Anthony · Location: Kingman, Arizona

Casing & Punctuation

  • Sentence case for headlines
  • UPPERCASE TRACKED for eyebrows
  • Square brackets for CTAs: [ Inquire Within ]
  • Em-dashes and → for flow

Writing Examples

✓ DO write like this:

"Your apps talk to each other. You don't have to."
"We answer the phone. In person. Every time."
"Repetitive tasks are for machines. Go take a real lunch break."
"Monitoring runs 24/7. If something breaks at 2am, that's my problem — not yours."
"Strategy is free. Execution is paid."
✗ DO NOT write like this:

"We facilitate seamless API-level ecosystem integration."
"We're passionate about nurturing your digital journey."
"Leverage scalable, user-centric digital solutions."
Anything that requires the reader to look up a word.

Marketing Voice Guidelines

✓ Marketing voice examples:

"Your site is turning away Spanish-speaking customers. Want to know why?"
"Machine translation is fine for a first draft. Public-facing? Not so much."
"Bilingual content isn't a bonus feature — it's respect."
"Most websites treat Spanish like an afterthought. Yours shouldn't."

When to Use Secondary Voice

  • Social media posts — Twitter, LinkedIn, Instagram captions
  • Email campaigns — Newsletter bodies, promotional blasts
  • Ad copy — Google Ads, Facebook, LinkedIn ads
  • Blog intros & conclusions — Story framing, calls to action
  • Testimonials & case studies — Client voices echoing brand tone
  • Promotional graphics — Pamali asset copy, social carousels

Target Audience

Who we're talking to. These are the small business owners, local entrepreneurs, and service professionals in the Southwest who know they have a problem but aren't sure where to start.

Small Local Business Owner

Pain: "My site is broken, but I don't know what's wrong or who to call."

Age: 35–55 · Tech: Moderate

Location: Mohave County & surrounding

Decision maker: Yes (usually sole decision maker)

Marketing Manager (Mid-size Biz)

Pain: "We need to expand to Spanish-speaking markets but don't know how to do it right."

Age: 28–42 · Tech: High

Budget: Has one; guards it closely

Decision maker: With CFO approval

Bilingual Community Leader

Pain: "Local businesses ignore Spanish speakers, then wonder why they can't sell."

Age: Any · Tech: Variable

Influence: High (recommends vendors)

Motivation: Community respect, trust

What They Care About

  • ✓ Being heard the first time — clear answers without jargon
  • ✓ Knowing costs upfront — no hidden fees or surprises
  • ✓ Doing it right once — not quick fixes that fail later
  • ✓ Actual human contact — someone who picks up the phone
  • ✓ Respecting their customers — proper localization, not an afterthought
  • ✓ Staying in business — solutions that actually convert

What They Don't Care About

  • ✗ Trendy design patterns or "cutting edge" tech
  • ✗ Case studies from companies 100x their size
  • ✗ Calls from salespeople during lunch
  • ✗ Technical explanations when they asked for a solution
  • ✗ Promises instead of guarantees

CMS Integration

How to connect this hub to your connected CMS and enable live updates without rebuilding the design system.

Option 1: Content Calendar Component (Recommended)

Use the reusable ContentCalendar.jsx component from the UI kit. It consumes JSON data directly from your CMS API and re-renders live when you update schedules.

// In your CMS page template: <x-import component="ContentCalendar" from="../../../ui_kits/website/ContentCalendar.jsx" data="{{ cms.contentCalendar }}" hint-size="100%,600px"></x-import> // CMS API returns: { "contentCalendar": [ { "day": 1, "week": 1, "category": "Local data", "topic": "Mohave County market", "hook": "...", "caption": "...", "cta": "...", "promptRef": "icon-001", "status": "scheduled", "scheduledDate": "2026-06-18" } // ...30 items ] }

Option 2: JSON Data Store (If No Real-time Needed)

Export the CSV as JSON, host it in your CMS assets folder, and import it statically. Update in CMS, re-export CSV → JSON, deploy. Good for weekly/monthly updates.

// data/content-calendar.json { "calendarEntries": [ { "day": 1, "week": 1, ... }, { "day": 2, "week": 1, ... } ], "lastUpdated": "2026-06-18T14:30:00Z" } // Import in template: <sc-for list="{{ calendar }}" as="entry">...</sc-for>

Option 3: Embedded CMS Editor (Advanced)

If your CMS has a rich editor UI, embed a content calendar table inline using your CMS's native table editor. Sync changes back to the API on save.

Implementation Steps

  1. Export CSV → JSON using the converter at uploads/da-master-social-copy.csv
  2. Create a CMS collection called "Content Calendar" with fields: day, week, category, topic, hook, caption, cta, promptRef, status, scheduledDate
  3. Populate initial 30 rows from the CSV data
  4. Set up API endpoint at /api/marketing/calendar that returns all entries sorted by day
  5. Mount the ContentCalendar.jsx component in your marketing page template, passing data="{{ calendar }}"
  6. Test live edits — update a calendar entry in the CMS, see it update in the component (no page refresh needed)

ContentCalendar.jsx API

Props: - data: Array of calendar entry objects - onEntryClick: (entry) => void (optional, for edit modals) - sortBy: 'day' | 'category' | 'status' (default: 'day') - showStatus: boolean (default: true) - showPromptRef: boolean (default: true) Example: <ContentCalendar data={calendarEntries} showStatus={true} onEntryClick={(entry) => openEditModal(entry)} sortBy="day" />
